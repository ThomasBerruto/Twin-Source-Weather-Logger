Twin-Source Weather Logger: Edge-to-Cloud IoT + Data Science

EdgeCloudOffline LoggingPython Analytics

What it does

  • Reads indoor temperature/humidity from a DHT22 sensor
  • Fetches outdoor weather for a ZIP code using wttr.in JSON
  • Publishes values to an Arduino IoT Cloud dashboard
  • Logs all measurements to an SD card CSV for later analysis

Protocols touched

  • Wi‑Fi → Internet connectivity
  • HTTP GET + JSON parsing (weather API)
  • NTP for timestamps
  • SPI (SD card)
  • I²C (RTC module)
  • IoT Cloud messaging (MQTT/SenML-style) + X.509 trust

CSV schema

date_time,date,hour,minutes,seconds,sensor_name,measurement,sensor_value,Units

Analytics ideas

  • Indoor vs outdoor deltas (bias/drift)
  • Rolling averages and daily seasonality
  • Correlation and lag analysis
  • Anomaly detection (jumps/outliers)
  • Comfort metrics (heat index / dew point)

Try it locally

pip install -r requirements.txt
python analysis_starter.py --csv data/sensor_log.csv
