🚀 New Project: Twin-Source Weather Logger: Edge-to-Cloud IoT + Data Science

I built an end-to-end IoT data pipeline that captures *local indoor conditions* and compares them to *live web weather* for the same ZIP code — then logs everything for analysis.

🧩 Stack
• Hardware: Wemos D1 R32 (ESP32-class) + data-logging shield (RTC + SD) + DHT22
• Connectivity: Wi‑Fi → Arduino IoT Cloud dashboard (real-time monitoring)
• Data sources: DHT22 (temp/humidity) + wttr.in JSON weather (ZIP-based)
• Persistence: SD card CSV (offline-first raw log)
• Analytics: Python (NumPy/Pandas/Seaborn) + Streamlit dashboard

🔁 Data Flow (high-level)
1) Read DHT22 temperature/humidity
2) Fetch web weather via HTTP GET (JSON)
3) Push values to Arduino IoT Cloud properties
4) Append timestamped rows to CSV on SD card

📈 What I’m analyzing next
• Indoor vs outdoor deltas (bias, drift)
• Daily/weekly seasonality
• Correlation between temp & humidity
• Anomaly detection (sensor glitches / sudden jumps)
• Comfort metrics (heat index / dew point)

✨ Key learnings
• Edge logging + cloud monitoring gives resilience and visibility
• Working across protocols (Wi‑Fi, HTTP, MQTT-style IoT messaging, SPI/I²C) makes the system feel “real”
